Activity 1 – Basic CSS Styles

Understanding CSS Fundamentals

CSS, or Cascading Style Sheets, controls the visual presentation of HTML elements. With just a few properties — color, text-align, font-size — you can dramatically transform how a webpage looks and feels.

Selectors let you target specific elements, while properties define what to change. This heading is blue and centered using color: blue and text-align: center. These paragraphs use font-size: 18px and color: gray, exactly as required by the activity.

CSS Properties Used

color: blue — Sets the text color of the heading to blue using a named CSS color keyword. Named colors like blue, green, gray, and red are built into every browser.

text-align: center — Centers the heading text horizontally within its container. Other values include left, right, and justify.
font-size: 18px — Sets paragraph text to 18 pixels, making it comfortable to read. color: gray applies a neutral tone to the paragraph text.

Activity 2 – Class and ID Selectors

CSS Selectors in Action

An ID selector (e.g. #title) targets a single unique element on the page. It is the most specific selector, meaning it overrides class and element rules when there is a conflict. Use it for elements that appear exactly once.

A class selector (e.g. .info) can be applied to many elements at once, making it perfect for reusable styles like this beige background, 16px font size, and comfortable padding applied to both paragraphs here.

Selector Reference

Selector: #title
Style: Green text, centered
This ID selector targets the one unique <h1> element and applies color: green and text-align: center.

Selector: .info
Style: Beige background, font-size 16px, padding
This class selector is applied to both paragraphs simultaneously, demonstrating how a single class rule can style multiple elements at once.

Full Stack Development — Class & ID Selectors

Full Stack Development

Full stack development covers both the client-facing front end and the server-side back end of a web application — plus the databases and infrastructure that tie them together. A full stack developer can build and ship a complete product independently.

This section applies the same CSS rules from Activity 2: #id selectors for green centered headings, and .info class for reusable beige blocks with 16px text and padding — submitted by Gimbert Ludovice under Prof. Rhea Lei Talavera.

Front End

The front end is everything a user sees and interacts with in the browser. Built with HTML for structure, CSS for styling, and JavaScript for interactivity. Modern frameworks like React, Vue, and Angular make building complex, dynamic UIs faster and more maintainable.

HTML5 CSS3 JavaScript React Vue

Responsive design ensures pages look great on any screen — critical for the Mobile & Website Development specialization. Flexbox, CSS Grid, and media queries let developers adapt layouts for mobile, tablet, and desktop.

Back End

The back end runs on a server and handles business logic, authentication, and database communication. Common languages include Node.js, Python, PHP, and Java. Frameworks like Express, Django, and Laravel provide structure and speed up development.

Node.js Python PHP Express Laravel

REST APIs connect the front end to the back end, sending and receiving JSON data over HTTP. This lets the browser request only the data it needs — no full-page reloads — resulting in faster, smoother web and mobile experiences.

Database

Databases store all persistent data — user accounts, posts, orders, and more. Relational databases like MySQL and PostgreSQL organize data in structured tables. NoSQL databases like MongoDB store flexible, document-based data suited to rapidly changing requirements.

MySQL PostgreSQL MongoDB Firebase SQLite

SQL (Structured Query Language) is the standard for relational databases. Commands like SELECT, INSERT, UPDATE, and DELETE give developers precise control over reading and writing data.

DevOps & Deployment

Once an application is built, it needs to be deployed so real users can access it. Platforms like Vercel, Netlify, and Railway make deploying apps straightforward. Docker packages an app and all its dependencies into a portable container.

Docker Vercel Netlify GitHub Actions CI/CD

Version control with Git and GitHub lets teams collaborate safely. CI/CD pipelines automatically test and deploy code on every push, reducing human error and accelerating releases.
